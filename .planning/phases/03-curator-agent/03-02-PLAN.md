<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Google Cloud Console → local filesystem | Service account JSON travels from browser download to `.secrets/` |
| local helper script → Google Sheets API | Authenticated request with service account credential |
| verification script → terminal output | Script must not leak credential fields into stdout/stderr/logs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Information disclosure | credential file contents leaking via verification script | mitigate | Acceptance criterion forbids `cat .secrets/...` or `grep .secrets/service-account` patterns in the script; only a `test -f` existence check is allowed |
| T-03-07 | Spoofing | service account share misconfiguration | mitigate | Task 1 resume-signal requires user confirmation that the service account email is shared on the specific Bank sheet with Editor role |
| T-03-08 | Elevation of privilege | service account granted project-wide roles unnecessarily | mitigate | Task 1 instructions explicitly state "Grant role: none required at the project level" — access is granted at sheet level only |
| T-03-09 | Tampering | wrong credential file placed at .secrets/service-account.json | mitigate | Task 1 acceptance verifies the JSON shape via `jq -e '.client_email and .private_key'`; Task 2 acceptance verifies the credential actually authenticates against the real Bank tab |
</threat_model>
